# 🧪 Directive: Usability Testing (SOP-PRO-04)

## 🎯 Primary Objective
Detect and eliminate cognitive friction in the Tadpole OS user experience. This directive ensures that the "Sovereign" interface is intuitive for both entry-level operators and expert architects.

---

## 🏗️ Testing Framework

### 1. Critical User Journeys (CUJ)
- **CUJ-01**: "First-Time Identity Setup" (Vault unlock to First Mission).
- **CUJ-02**: "Swarm Pulse Coordination" (Detaching visualizer to analyzing logs).
- **CUJ-03**: "Governance Override" (Human-in-the-loop approval of a shell tool).

### 2. Observational Methodology
- **Action**: Conduct "Think-Aloud" sessions with internal or community testers.
- **Metric**: Track "Time-to-Completion" and "Error-Rate" for each CUJ.

---

## 🛠️ Analysis SOP

### 1. Friction Discovery
- **Focus**: Where did the user pause? Which icons were unclear? 
- **Action**: Log these as "Cognitive Blocks" in the `usability_report_[DATE].md`.

### 2. Resolution Path
- **Action**: Assign a "Friction Score" (1-10). Scores > 7 require an immediate `design_system_update.md` or a tutorial snippet addition to `GETTING_STARTED.md`.

---

## 📊 Reporting Protocol
Results must be synthesized into the bi-monthly `product_sync.md`. Focus on "High-Leverage" UI fixes that improve the overall efficiency of the sovereign swarm.
