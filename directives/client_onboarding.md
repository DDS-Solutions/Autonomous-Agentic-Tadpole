# 🤝 Directive: Client Onboarding (SOP-HR-03)

## 🎯 Primary Objective
Provide a seamless, "White-Glove" entry experience for new organizations joining the Tadpole OS ecosystem. Focus on "Time-to-Value" (TTV) for the sovereign swarm.

---

## 🏗️ Onboarding Roadmap

### 1. Workspace Initialization
- **Action**: Create a dedicated `cluster_id` and filesystem sandbox.
- **Check**: Ensure `base_dir` resolution is correct for the host OS.

### 2. Identity Forge
- **Role Assignment**: Initialize a "Cluster CEO" and 3 specialists (Admin, Developer, Marketing).
- **Authentication**: Walk the user through the `Neural Vault` setup and key injection.

### 3. Governance Tuning
- **Budget**: Initialize the `DEFAULT_AGENT_BUDGET_USD` and set up the `Budget Guard` thresholds.
- **Privacy**: Set the `PRIVACY_MODE` based on the client's air-gap requirements.

---

## 🛠️ Initial Mission SOP

### 1. The "First Light" Task
- **Action**: Assign a simple research task (e.g., "Audit local documentation") to the new cluster.
- **Goal**: Verify end-to-end communication from the UI to the Engine and back.

### 2. Walkthrough Generation
- **Deliverable**: `reports/ONBOARDING_WALKTHROUGH.md`. Include specific instructions for using the `Sovereign Dashboard`.

---

## 🚦 Success Criteria
Successful onboarding is confirmed when the client's first mission reaches "Finished" status with zero security violations and a manually signed-off oversight gate.
